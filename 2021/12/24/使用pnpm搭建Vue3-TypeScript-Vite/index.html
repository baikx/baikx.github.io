背景

最近我终于想起来我搭建过一个博客，并且想更新点内容上去，但是当我更新了第一篇Post not found: canvas实现头像裁剪缩放 canvas实现头像裁剪缩放之后我发现了一个问题。这篇博客有很多内容涉及到了图片，但是都放在博客里会无故扩充博客的大小，因此我准备搞一个 CDN 来分发，在调研了一圈之后，我发现买一个云服务器来放我本就不多的内容实在太浪费了。所以，我准备白嫖一个CDN，那么肯定有朋友就问了，怎么白嫖 CDN 呢，这里留个坑吧，过两天我更新一篇怎么白嫖 CDN 的文章。那么当我白嫖了 CDN 之后，我又又又发现了一个问题，白嫖的 CDN 不能放太大的文件，而且整个包的大小也要控制在 50MB 以内。这就让人很难受了！我的上一篇内容还没写完，里面的图片就已经占了 26MB 。那我以后的其他文章还怎么插图，因此我就突发其想，准备搞一个图片压缩工具，把图片压缩一下，毕竟写的内容才是重点，图片只是辅助理解，只要能看的清就好（我不会告诉你我是因为买不起 CDN 才这么干的）。好了话不多说，下面开始详细的教程吧。

安装pnpm、vite

为什么使用pnpm呢？

主要是因为当我们使用其他的包管理工具，比如 npm 或 yran时，假设我们有很多个项目使用了同一个依赖，就会有很多份这个依赖的副本存在硬盘上。使用 pnpm 时，依赖会存储在内容可寻址的存储上。

那么这样做解决了什么痛点呢？

  1. 如果用到了某个依赖的不同版本，就只会把有差异的文件添加到仓库。举个🌰 ，如果项目中的一个依赖包有50个文件，但是最新版本只更新了一个文件，那么 pnpm update 时只会向存储中心额外添加这个更新的文件，而不是复制整个新版本包的内容，大大节省了存储空间。
  2. 所有的文件会存储在硬盘的某一个位置。当你安装依赖包时，包里的文件会硬连接到这个位置，不会占用额外的磁盘空间。允许你跨项目的共享同一个版本的依赖。比如你的项目里添加了新的依赖，在其他项目中也使用了，那么这个项目在安装依赖时，就会直接复用其他项目中的依赖，节省磁盘空间，并且大幅提升安装速度，尤其是因为某些众所周知的原因，有些库时不时安装不上的情况可以大幅缓解。

pnpm会创建非扁平化的 node_modules 文件夹，当你使用 npm 等安装依赖时，你会发现你的 node_modules 文件夹下有很多不是你自己进行安装的依赖，因为所有的依赖都会被提升到模块的根目录中。pnpm使用软链的方式将项目的依赖添加进模块文件夹的根目录。

可以直接使用 pnpm 创建 vite 项目:

在命令行输入 pnpm create vite 回车后，起一个名称，默认是 vite-project 修改之后，回车，进行可视化库选择，使用上下按键进行选择，选中 vue 回车。进行语言选择，vue 是 JavaScript 语言进行开发，选择 vue-ts 使用 TypeScript语言进行开发。至此项目已经搭建完成了。
