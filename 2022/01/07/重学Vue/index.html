重学Vue

按照Vue文档进行梳理，补充和总结容易遗漏但是非常重要的点。

应用&组件实例

createApp 创建应用实例

创建的应用实例是用来在应用中注册全局组件的。

根组件

  • 传递给 createApp 的选项用于配置根组件，挂载应用时，该组件被用作渲染的起点。
  • 应用通过 mount 挂载到一个 DOM 元素中。
  • mount 不返回应用本身，返回根组件实例。
  • 每个组件由自己组件实例的 vm ( ViewModel 的缩写 )。
  • 应用中的所有组件实例共享同一个应用实例。
  • 根组件和其他组件的配置选项、组件实例行为都一样。

组件实例

组件实例的所有 property 无论如何定义都可以在组件的模板中访问。
一些内置的 property 避免和用户定义的 property 名冲突，使用 $ 前缀。

生命周期钩子

组件在创建时会经过初始化过程，在这个过程中运行一些生命周期钩子函数，给用户在不用阶段添加自己的代码。

模板语法

  • Vue.js 使用了基于 HTML 的模板语法。
  • 允许开发者声明式的将 DOM 绑定至底层组件实例的数据。
  • Vue.js 的模板都是合法的 HTML，能被遵循规范的浏览器和 HTML 解析器解析。
  • Vue.js 将模板编译成虚拟 DOM渲染函数。
  • 结合响应性系统，计算最少需要重新渲染多少组件，把 DOM 操作次数减到最少。

插值

文本插值{{}}指令：数据绑定的常见形式就是使用 Mustache (双大括号) 语法的文本插值，双括号会将数据解释为普通文本。

Mustache 标签将会被替代为对应组件实例中 msg property 的值。

无论何时，绑定的组件上 msg property 发生了改变，插值处的内容都会更新。

一次性插值 v-once 指令：数据改变时，插值处的内容不会更新。

这个命令会影响这个节点的其他数据绑定。

原始 HTML 插值 v-html 指令：替换数据为原始 html 代码。

不能使用v-html进行复合局部模板，Vue不是基于字符串的模板引擎。组件更适合作为可重用和可复用的基本单位。

Attritube 绑定 v-bind 指令：Mustache 语法不能在 HTML Attribute 中使用。

  • 绑定值为 null、 undefined 时将不会包含在渲染的元素上。
  • 对于布尔 attribute（存在即为true） 绑定值为 truthy 或 空字符串（与[attribute]=""保持一致） 就将该 attribute 包含在内，其他 falsy 的值，该 attribute 将被省略。

在指令或 Mustache 中使用 JavaScript 表达式：表达式会在当前活动实例的数据作作用域下作为 JavaScript 被解析。

每个绑定都只能包含单个表达式

附上源码及解析：

指令

  • 指令是带有 v- 前缀的特殊 attribute。
  • 指令 attribute 的值预期是单个 JavaScript 表达式（v-for、v-on例外）。
  • 表达式的值改变时，产生连带影响，响应式的作用与 DOM 。
参数

一些指令能接收一个参数，在指令名称后以冒号加参数名表示。

动态参数

可以在参数中使用 JavaScript 表达式，使用方括号括起来

  • 动态参数会求出一个字符串，null例外。
  • null值可用于显式的移除绑定。
  • 任何其他非字符串类型的值会触发一个警告。
  • 动态参数表达式有语法约束，某些字符（空格、引号），放在 HTML attribute 名里时无效的。
  • 避免使用大写自负命名，浏览器会把attribute全部强制转为小写。
修饰符

修饰符以半角句号 . 指明的特殊后缀，用于指出一个指令应该以特殊方式绑定。

缩写

  • v-前缀作为视觉提示，用来识别模板中 Vue 特定的 attribute。
  • Vue 为 v-bind（简写 :参数） 和 v-on（简写 @参数） 两个常用的指令提供了特定简写。
