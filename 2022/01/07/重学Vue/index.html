重学Vue

按照Vue文档进行梳理，补充和总结容易遗漏但是非常重要的点。

应用&组件实例

createApp 创建应用实例

创建的应用实例是用来在应用中注册全局组件的。

根组件

  • 传递给 createApp 的选项用于配置根组件，挂载应用时，该组件被用作渲染的起点。
  • 应用通过 mount 挂载到一个 DOM 元素中。
  • mount 不返回应用本身，返回根组件实例。
  • 每个组件由自己组件实例的 vm ( ViewModel 的缩写 )。
  • 应用中的所有组件实例共享同一个应用实例。
  • 根组件和其他组件的配置选项、组件实例行为都一样。

组件实例

组件实例的所有 property 无论如何定义都可以在组件的模板中访问。
一些内置的 property 避免和用户定义的 property 名冲突，使用 $ 前缀。

生命周期钩子

组件在创建时会经过初始化过程，在这个过程中运行一些生命周期钩子函数，给用户在不用阶段添加自己的代码。

模板语法

  • Vue.js 使用了基于 HTML 的模板语法。
  • 允许开发者声明式的将 DOM 绑定至底层组件实例的数据。
  • Vue.js 的模板都是合法的 HTML，能被遵循规范的浏览器和 HTML 解析器解析。
  • Vue.js 将模板编译成虚拟 DOM渲染函数。
  • 结合响应性系统，计算最少需要重新渲染多少组件，把 DOM 操作次数减到最少。

插值

文本插值{{}}指令：数据绑定的常见形式就是使用 Mustache (双大括号) 语法的文本插值，双括号会将数据解释为普通文本。

Mustache 标签将会被替代为对应组件实例中 msg property 的值。

无论何时，绑定的组件上 msg property 发生了改变，插值处的内容都会更新。

一次性插值 v-once 指令：数据改变时，插值处的内容不会更新。

这个命令会影响这个节点的其他数据绑定。

原始 HTML 插值 v-html 指令：替换数据为原始 html 代码。

不能使用v-html进行复合局部模板，Vue不是基于字符串的模板引擎。组件更适合作为可重用和可复用的基本单位。

Attritube 绑定 v-bind 指令：Mustache 语法不能在 HTML Attribute 中使用。

  • 绑定值为 null、 undefined 时将不会包含在渲染的元素上。
  • 对于布尔 attribute（存在即为true） 绑定值为 truthy 或 空字符串（与[attribute]=""保持一致） 就将该 attribute 包含在内，其他 falsy 的值，该 attribute 将被省略。

在指令或 Mustache 中使用 JavaScript 表达式：表达式会在当前活动实例的数据作作用域下作为 JavaScript 被解析。

每个绑定都只能包含单个表达式

附上源码及解析：

指令

  • 指令是带有 v- 前缀的特殊 attribute。
  • 指令 attribute 的值预期是单个 JavaScript 表达式（v-for、v-on例外）。
  • 表达式的值改变时，产生连带影响，响应式的作用与 DOM 。
参数

一些指令能接收一个参数，在指令名称后以冒号加参数名表示。

动态参数

可以在参数中使用 JavaScript 表达式，使用方括号括起来

  • 动态参数会求出一个字符串，null值可用于显式的移除绑定。
  • 任何其他非字符串类型的值会触发一个警告。
  • 动态参数表达式有语法约束，某些字符（空格、引号），放在 HTML attribute 名里时无效的。
  • 避免使用大写字符命名，浏览器会把attribute全部强制转为小写。
修饰符

修饰符以半角句号 . 指明的特殊后缀，用于指出一个指令应该以特殊方式绑定。

缩写

  • v-前缀作为视觉提示，用来识别模板中 Vue 特定的 attribute。
  • Vue 为 v-bind（简写 :参数） 和 v-on（简写 @参数） 两个常用的指令提供了特定简写。

Data Property 和方法

Data Property

  • 组件的 data 选项是一个函数。
  • Vue 会在创建新的组件实例的过程中调用此函数，返回一个对象。
  • Vue 通过响应式系统将其包裹，以 $data 形式存储在组件实例中。
  • 该对象的任何顶级 property 也会通过组件实例暴露出来。
  • 实例 property 仅在实例首次创建时被添加，确保在data函数返回的对象中。
  • 对尚未提供所需值的 property 使用 null、undefined或其他占位置。

直接将不包含在 data 中的新 property 添加到组件实例是可行的，但该property不在背后响应式 $data 对象内，所以 Vue 的响应式系统不会自动跟踪它

Vue 使用 $ 前缀通过组件实例暴露自己的内置API，内部property保留 _ 前缀，使用时需要避免

方法

  • 使用 methods 选项向组件实例添加方法。
  • methods 是一个包含所需方法的对象。
  • Vue 自动为 methods 绑定 this，定义时应避免使用箭头函数。
  • methods 和组件实例的其他 property 一样可以在组件的模板中被访问。
  • 使用生命周期函数替换，从模板调用的有任何副作用的方法（更改数据、触发异步进程等）。
  • 可在在模板支持 JavaScript 表达式的任何地方调用方法。

防抖和节流

Vue没有内置支持的防抖和节流，使用其他第三方库来实现。

  • 组件仅使用一次时，可以在 methods 中直接应用。
  • 可复用组件在生命周期钩子的 created 里添加防抖函数实现彼此独立。

计算属性和侦听器

模版内的表达式仅用于简单计算，放入过多逻辑会导致模版过重且难以维护。使用计算属性处理任何包含响应式数据的复杂逻辑。

  • 像普通属性一样将数据绑定到模板中的计算属性
  • 计算属性的 getter 函数没有副作用，更易于测试和理解。
  • 计算属性将基于它们的响应式依赖关系缓存。
  • 计算属性会在相关响应式依赖发生改变时重新求值。
  • 不希望有缓存时，使用 method 来代替。
  • 计算属性默认只有 getter，需要时可自行添加 setter

侦听器

  • watch 是Vue提供的自定义侦听器，更加通用的响应数据变化。
  • watch 在处理数据变化时执行异步或开销较大的操作时最优。
  • 还可以使用命令式的 vm.$wacth API

计算属性 vs 侦听器

Vue提供更为通用的侦听属性来观察和响应当前活动的实例上的数据变动。
当有一些数据需要随着其他数据变动而变动时，watch很容易被滥用。

class 与 style 绑定

在 v-bind 用于 class 和 style 时，Vue.js 进行了专门的增强。表达式结果类型可以是字符串(String)、对象(Object)或数组(Array)。

绑定 HTML Class

  • :class 指令可以与普通 class attribute 共存。

对象语法：我们可以传给 :class (v-bind 的简写)一个对象，动态切换class。

数组语法：把一个数组传给:class，应用一个 class 列表。

在组件上使用

在带有单个根元素的自定义组件上使用 calss attribute 时，这些 class 将被添加到该元素中，且不会覆盖此元素上现有的 class 。

多个根元素需定义接收calss的部分，可以使用$attrs组件property进行操作

绑定内联样式

对象语法：:style 的对象语法非常直观，看起来像css。但其实是一个JavaScript 对象。

  • CSS property 名可以用驼峰式(camelCase)或短横线分隔(kebab-case，用引号括起来)命名

数组语法：:style 的数组可以将多个样式对象应用到同一个元素上。

  • Vue在 :style 中使用需要一个 **vendor profix(浏览器引擎前缀)**的css property时，将自动侦测并添加相应的前缀。
  • :style 可以为style绑定中的property提供一个包含多个值的数组，常用于提供多个带前缀的值。
